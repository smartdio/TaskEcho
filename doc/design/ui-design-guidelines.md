# TaskEcho UI 设计规范

## 1. 概述

### 1.1 文档目的

本文档定义 TaskEcho 系统的 UI 设计规范，确保整个应用具有一致的视觉风格和用户体验。设计规范以移动端（手机和平板）为主要适配目标，桌面端为次要适配目标。

### 1.2 设计理念

- **移动优先**：优先考虑移动端体验，确保在手机和平板上的最佳表现
- **扁平化设计**：采用扁平化设计风格，避免过度装饰和阴影效果
- **信息层次**：通过视觉层次清晰呈现信息，帮助用户快速理解内容
- **自然色彩**：使用自然、柔和的色彩，避免过于鲜艳或刺眼的颜色
- **触摸友好**：所有交互元素针对触摸屏优化，确保易于点击和操作

### 1.3 适用范围

本规范适用于 TaskEcho 系统的所有界面设计，包括：
- 页面布局
- 组件设计
- 交互设计
- 响应式适配

## 2. 设备适配策略

### 2.1 设备优先级

| 优先级 | 设备类型 | 屏幕宽度范围 | 主要考虑因素 |
|--------|---------|-------------|-------------|
| **P0** | 手机 | < 768px | 触摸操作、单列布局、内容精简 |
| **P1** | 平板 | 768px - 1024px | 触摸操作、多列布局、内容扩展 |
| **P2** | 桌面 | > 1024px | 鼠标操作、多列布局、内容完整 |

### 2.2 响应式断点

基于 Tailwind CSS 标准断点，结合移动优先策略：

| 断点 | 最小宽度 | 设备类型 | 布局策略 |
|------|---------|---------|---------|
| **默认** | < 640px | 小屏手机 | 单列布局，紧凑间距 |
| **sm** | 640px | 大屏手机 | 单列布局，标准间距 |
| **md** | 768px | 平板（竖屏） | 双列布局，扩展间距 |
| **lg** | 1024px | 平板（横屏）/小桌面 | 多列布局，完整功能 |
| **xl** | 1280px | 桌面 | 多列布局，最大宽度限制 |

### 2.3 布局原则

- **移动端**：单列布局为主，垂直滚动，内容全宽显示
- **平板端**：可适当使用双列布局，保持触摸友好
- **桌面端**：多列布局，最大宽度限制（1280px），居中显示

## 3. 色彩系统

### 3.1 设计原则

- **自然柔和**：使用低饱和度的自然色彩
- **对比清晰**：确保文字和背景有足够的对比度（WCAG AA 标准：4.5:1）
- **语义化**：颜色具有明确的语义含义（成功、警告、错误等）

### 3.2 主色调

| 颜色名称 | HSL 值 | 用途 | Tailwind 类名 |
|---------|--------|------|--------------|
| **背景色** | `hsl(0, 0%, 100%)` | 页面主背景 | `bg-white` |
| **次要背景** | `hsl(0, 0%, 98%)` | 卡片、区块背景 | `bg-gray-50` |
| **主文字** | `hsl(222, 47%, 11%)` | 主要文字内容 | `text-gray-900` |
| **次要文字** | `hsl(215, 16%, 47%)` | 辅助文字、说明 | `text-gray-600` |
| **边框** | `hsl(214, 32%, 91%)` | 分割线、边框 | `border-gray-200` |

### 3.3 功能色彩

| 状态 | HSL 值 | 用途 | Tailwind 类名 |
|------|--------|------|--------------|
| **主要操作** | `hsl(221, 83%, 53%)` | 主要按钮、链接 | `bg-blue-600` / `text-blue-600` |
| **成功** | `hsl(142, 76%, 36%)` | 成功状态、完成 | `bg-green-600` / `text-green-600` |
| **警告** | `hsl(38, 92%, 50%)` | 警告信息、待处理 | `bg-yellow-500` / `text-yellow-600` |
| **错误** | `hsl(0, 84%, 60%)` | 错误状态、失败 | `bg-red-500` / `text-red-600` |
| **信息** | `hsl(199, 89%, 48%)` | 信息提示 | `bg-cyan-500` / `text-cyan-600` |

### 3.4 任务状态色彩映射

| 任务状态 | 颜色 | 说明 |
|---------|------|------|
| **Pending** | `yellow-500` | 待处理，使用警告色 |
| **Running** | `blue-500` | 运行中，使用主色调 |
| **Done** | `green-600` | 已完成，使用成功色 |
| **Error** | `red-500` | 错误，使用错误色 |
| **Cancelled** | `gray-500` | 已取消，使用中性色 |

### 3.5 色彩使用规范

#### 背景色使用

- **页面背景**：使用主背景色（白色）
- **卡片背景**：使用次要背景色（`bg-gray-50`），与页面背景形成轻微对比
- **悬停状态**：使用 `hover:bg-gray-100` 提供交互反馈

#### 文字颜色使用

- **标题文字**：使用主文字颜色（`text-gray-900`），确保高对比度
- **正文文字**：使用主文字颜色（`text-gray-900`）
- **辅助文字**：使用次要文字颜色（`text-gray-600`），如时间、说明等
- **禁用文字**：使用 `text-gray-400`

#### 边框颜色使用

- **分割线**：使用边框颜色，轻微可见即可
- **卡片边框**：仅在需要时使用，避免过度使用
- **输入框边框**：使用边框颜色，聚焦时使用主色调

## 5. 间距系统

### 5.1 间距原则

- **一致性**：使用统一的间距系统，避免随意设置
- **节奏感**：间距遵循 4px 或 8px 的倍数规则
- **响应式**：移动端使用较小间距，平板和桌面端适当增大

### 5.2 基础间距单位

基于 Tailwind CSS 的间距系统（1 单位 = 4px）：

| Tailwind 类名 | 实际值 | 使用场景 |
|--------------|--------|---------|
| `space-1` | 4px | 紧密元素间距 |
| `space-2` | 8px | 图标与文字间距 |
| `space-3` | 12px | 小元素间距 |
| `space-4` | 16px | 标准元素间距（移动端） |
| `space-5` | 20px | 中等元素间距（平板端） |
| `space-6` | 24px | 大元素间距（桌面端） |
| `space-8` | 32px | 区块间距 |
| `space-10` | 40px | 大区块间距 |
| `space-12` | 48px | 页面区块间距 |

### 5.3 内边距（Padding）

#### 移动端内边距

| 组件类型 | 内边距 | Tailwind 类名 | 使用场景 |
|---------|--------|--------------|---------|
| **页面容器** | 16px | `px-4` | 页面左右边距 |
| **卡片** | 16px | `p-4` | 卡片内边距 |
| **按钮** | 12px 16px | `py-3 px-4` | 标准按钮 |
| **输入框** | 12px 16px | `py-3 px-4` | 文本输入框 |
| **列表项** | 12px 16px | `py-3 px-4` | 列表项内边距 |

#### 平板端内边距

| 组件类型 | 内边距 | Tailwind 类名 |
|---------|--------|--------------|
| **页面容器** | 20px | `md:px-5` |
| **卡片** | 20px | `md:p-5` |
| **按钮** | 14px 20px | `md:py-3.5 md:px-5` |
| **输入框** | 14px 20px | `md:py-3.5 md:px-5` |

#### 桌面端内边距

| 组件类型 | 内边距 | Tailwind 类名 |
|---------|--------|--------------|
| **页面容器** | 24px | `lg:px-6` |
| **卡片** | 24px | `lg:p-6` |
| **按钮** | 16px 24px | `lg:py-4 lg:px-6` |

### 5.4 外边距（Margin）

| 使用场景 | 移动端 | 平板端 | 桌面端 | Tailwind 类名 |
|---------|--------|--------|--------|--------------|
| **元素间距** | 16px | 20px | 24px | `mb-4 md:mb-5 lg:mb-6` |
| **区块间距** | 24px | 32px | 40px | `mb-6 md:mb-8 lg:mb-10` |
| **页面区块** | 32px | 40px | 48px | `mb-8 md:mb-10 lg:mb-12` |

### 5.5 间距使用规范

- **垂直间距**：优先使用 `space-y-*` 或 `mb-*` 设置垂直间距
- **水平间距**：优先使用 `space-x-*` 或 `mr-*` 设置水平间距
- **响应式间距**：使用响应式类名实现不同设备的间距适配
- **避免负边距**：尽量避免使用负边距，保持布局清晰

## 6. 组件设计规范

### 6.1 按钮（Button）

#### 设计原则

- **触摸友好**：最小触摸区域 44×44px（移动端）
- **视觉清晰**：按钮样式明确，易于识别
- **状态反馈**：提供明确的悬停、激活、禁用状态

#### 尺寸规范

| 尺寸 | 移动端 | 平板端 | 桌面端 | Tailwind 类名 |
|------|--------|--------|--------|--------------|
| **小** | 36px | 38px | 40px | `h-9 md:h-9.5 lg:h-10` |
| **标准** | 44px | 44px | 44px | `h-11` |
| **大** | 48px | 48px | 48px | `h-12` |

#### 样式变体

| 变体 | 样式 | 使用场景 |
|------|------|---------|
| **主要按钮** | 主色调背景，白色文字 | 主要操作 |
| **次要按钮** | 透明背景，主色调边框和文字 | 次要操作 |
| **文字按钮** | 透明背景，主色调文字 | 辅助操作 |
| **危险按钮** | 红色背景，白色文字 | 删除、危险操作 |

#### 代码示例

```tsx
// 主要按钮
<Button className="h-11 bg-blue-600 text-white hover:bg-blue-700">
  提交
</Button>

// 次要按钮
<Button variant="outline" className="h-11 border-blue-600 text-blue-600">
  取消
</Button>

// 文字按钮
<Button variant="ghost" className="h-11 text-blue-600">
  查看详情
</Button>
```

### 6.2 卡片（Card）

#### 设计原则

- **扁平化**：避免过度阴影，使用轻微边框或背景色区分
- **内容清晰**：合理的内边距和间距，确保内容易读
- **响应式**：移动端全宽，平板和桌面端可多列布局

#### 样式规范

| 属性 | 移动端 | 平板端 | 桌面端 |
|------|--------|--------|--------|
| **内边距** | 16px | 20px | 24px |
| **圆角** | 8px | 8px | 12px |
| **边框** | 1px solid | 1px solid | 1px solid |
| **背景** | 次要背景色 | 次要背景色 | 次要背景色 |

#### 代码示例

```tsx
<Card className="p-4 md:p-5 lg:p-6 rounded-lg md:rounded-lg lg:rounded-xl border border-gray-200 bg-gray-50">
  <CardHeader>
    <CardTitle>标题</CardTitle>
  </CardHeader>
  <CardContent>
    <p>内容</p>
  </CardContent>
</Card>
```

### 6.3 输入框（Input）

#### 设计原则

- **触摸友好**：最小高度 44px，易于点击
- **视觉反馈**：清晰的聚焦状态和错误状态
- **标签清晰**：标签和输入框关联明确

#### 尺寸规范

| 属性 | 移动端 | 平板端 | 桌面端 |
|------|--------|--------|--------|
| **高度** | 44px | 44px | 44px |
| **内边距** | 12px 16px | 14px 20px | 16px 24px |
| **字体大小** | 16px | 16px | 16px |

#### 状态样式

| 状态 | 边框颜色 | 说明 |
|------|---------|------|
| **默认** | 边框色 | 正常状态 |
| **聚焦** | 主色调 | 聚焦时高亮 |
| **错误** | 错误色 | 验证失败 |
| **禁用** | 边框色（降低透明度） | 禁用状态 |

### 6.4 列表（List）

#### 设计原则

- **触摸友好**：列表项最小高度 44px
- **视觉分隔**：使用分割线或间距区分列表项
- **信息层次**：重要信息突出，次要信息弱化

#### 列表项规范

| 属性 | 移动端 | 平板端 | 桌面端 |
|------|--------|--------|--------|
| **最小高度** | 44px | 44px | 48px |
| **内边距** | 12px 16px | 14px 20px | 16px 24px |
| **分割线** | 1px solid | 1px solid | 1px solid |

### 6.5 徽章（Badge）

#### 设计原则

- **尺寸适中**：不影响主要内容阅读
- **颜色语义**：使用功能色彩表示不同状态
- **圆角处理**：使用圆角，避免尖锐

#### 尺寸规范

| 尺寸 | 高度 | 内边距 | Tailwind 类名 |
|------|------|--------|--------------|
| **小** | 20px | 4px 8px | `h-5 px-2` |
| **标准** | 24px | 6px 12px | `h-6 px-3` |
| **大** | 28px | 8px 16px | `h-7 px-4` |

## 7. 触摸交互规范

### 7.1 触摸目标尺寸

| 元素类型 | 最小尺寸 | 推荐尺寸 | 说明 |
|---------|---------|---------|------|
| **按钮** | 44×44px | 48×48px | 主要操作按钮 |
| **图标按钮** | 44×44px | 48×48px | 图标按钮 |
| **链接** | 44px（高度） | 48px（高度） | 文字链接 |
| **列表项** | 44px（高度） | 48px（高度） | 可点击列表项 |
| **输入框** | 44px（高度） | 48px（高度） | 输入框高度 |

### 7.2 触摸反馈

#### 视觉反馈

- **按下状态**：使用 `active:` 状态，降低透明度或改变背景色
- **悬停状态**：桌面端使用 `hover:` 状态（移动端不适用）
- **聚焦状态**：使用 `focus-visible:` 状态，显示聚焦环

#### 代码示例

```tsx
<Button className="active:opacity-80 hover:bg-blue-700 focus-visible:ring-2 focus-visible:ring-blue-500">
  按钮
</Button>
```

### 7.3 手势支持

| 手势 | 用途 | 实现方式 |
|------|------|---------|
| **点击** | 主要交互方式 | 标准点击事件 |
| **长按** | 上下文菜单（可选） | `onLongPress` |
| **滑动** | 列表滚动 | 原生滚动 |
| **下拉刷新** | 数据刷新（可选） | 自定义实现 |

### 7.4 交互优化

- **避免双击缩放**：设置 `touch-action: manipulation`
- **快速响应**：交互反馈延迟 < 100ms
- **防误触**：重要操作使用确认机制
- **加载状态**：长时间操作显示加载指示器

## 8. 视觉层次

### 8.1 层次原则

- **重要性排序**：重要信息更突出，次要信息更弱化
- **视觉权重**：通过大小、颜色、间距建立视觉权重
- **分组清晰**：相关内容分组，无关内容分离

### 8.2 层次建立方法

#### 通过大小建立层次

- **标题**：使用更大的字体和更粗的字重
- **正文**：使用标准字体大小
- **辅助信息**：使用较小的字体和较淡的颜色

#### 通过颜色建立层次

- **主要信息**：使用主文字颜色，高对比度
- **次要信息**：使用次要文字颜色，中等对比度
- **辅助信息**：使用更淡的颜色，低对比度

#### 通过间距建立层次

- **紧密相关**：使用较小间距（8-12px）
- **相关**：使用标准间距（16-24px）
- **无关**：使用较大间距（32-48px）

### 8.3 信息架构

#### 页面层级

```
页面标题（最大、最粗）
  └─ 区块标题（中等、粗体）
      └─ 内容标题（小、中等粗细）
          └─ 正文内容（标准、常规）
              └─ 辅助信息（小、淡色）
```

#### 卡片层级

```
卡片标题（粗体、主文字色）
  └─ 卡片内容（常规、主文字色）
      └─ 卡片元数据（小字、次要文字色）
```

## 9. 动画和过渡

### 9.1 动画原则

- **自然流畅**：使用自然的缓动函数
- **适度使用**：避免过度动画，影响性能
- **有意义**：动画应传达信息，而非装饰

### 9.2 过渡时长

| 动画类型 | 时长 | 使用场景 |
|---------|------|---------|
| **快速** | 150ms | 悬停、聚焦状态变化 |
| **标准** | 200-300ms | 颜色、透明度变化 |
| **慢速** | 400-500ms | 布局变化、页面切换 |

### 9.3 缓动函数

| 缓动函数 | Tailwind 类名 | 使用场景 |
|---------|--------------|---------|
| **ease-in-out** | `ease-in-out` | 标准过渡（默认） |
| **ease-out** | `ease-out` | 进入动画 |
| **ease-in** | `ease-in` | 退出动画 |

### 9.4 常用动画

#### 淡入淡出

```tsx
<div className="transition-opacity duration-200 ease-in-out opacity-0 hover:opacity-100">
  内容
</div>
```

#### 颜色过渡

```tsx
<Button className="transition-colors duration-200 ease-in-out bg-blue-600 hover:bg-blue-700">
  按钮
</Button>
```

#### 缩放反馈

```tsx
<Button className="transition-transform duration-150 ease-out active:scale-95">
  按钮
</Button>
```

### 9.5 性能优化

- **使用 CSS 动画**：优先使用 CSS `transition` 和 `transform`
- **避免重排**：使用 `transform` 和 `opacity` 而非改变布局属性
- **减少动画**：移动端可适当减少动画，提升性能
- **禁用动画**：支持 `prefers-reduced-motion` 媒体查询

### 10.2 图标尺寸

## 13. 无障碍性（A11y）

### 13.1 基本原则

- **语义化 HTML**：使用正确的 HTML 标签
- **键盘导航**：所有交互元素支持键盘操作
- **屏幕阅读器**：提供适当的 ARIA 属性
- **颜色对比度**：确保文字和背景对比度符合 WCAG AA 标准

### 13.2 实践规范

#### 语义化标签

```tsx
// ✅ 正确
<header>
  <nav>
    <button aria-label="设置">设置</button>
  </nav>
</header>

// ❌ 错误
<div>
  <div>
    <div onClick={handleClick}>设置</div>
  </div>
</div>
```

#### ARIA 属性

```tsx
<button
  aria-label="删除项目"
  aria-describedby="delete-help-text"
>
  删除
</button>
<p id="delete-help-text">此操作不可撤销</p>
```

#### 键盘导航

```tsx
<button
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      handleClick()
    }
  }}
>
  按钮
</button>
```
